# Неделя 6

## Подключение PostgreSQL: миграции и CRUD-слой для электронного журнала

### Добавление goose-миграции схемы БД

На этой неделе в проект добавлена первая полноценная миграция через Goose:

- `migrations/20260412152000_init_parser_schema.sql`

Миграция содержит:
- `Up`-часть: создание таблиц и первичное наполнение справочника `control_types`
- `Down`-часть: удаление таблиц в корректном порядке по зависимостям

Созданы таблицы:
- `lecterns` — кафедры
- `control_types` — типы контроля (экзамен/зачет и т.д.)
- `subjects` — учебные дисциплины
- `subject_metrics` — метрики дисциплины (часы, ЗЕТ)
- `semester_load` — распределение нагрузки по семестрам
- `subject_controls` — формы контроля по семестрам
- `teachers` — преподаватели
- `groups` — учебные группы
- `students` — студенты

Зачем это нужно:
- схема БД теперь управляется версионно
- разворачивание одинаковой структуры на dev/test/prod становится предсказуемым
- проект готов к дальнейшему росту без ручного SQL в продакшене

### Подключение слоя работы с БД

Добавлен пакет `internal/db`, который инкапсулирует работу с PostgreSQL через `pgxpool`.

Ключевые изменения:
- `Store` с единым подключением к БД и доступом ко всем репозиториям
- `Ping` при старте для ранней проверки доступности БД
- подключение через `DB_DSN` в конфиге
- инициализация подключения в `cmd/server/main.go`

Практический эффект:
- приложение может работать с постоянным хранилищем
- доступ к данным централизован в одном слое
- проще подключать новые API-обработчики без дублирования SQL

### Реализованный CRUD и операции по сущностям

Для электронного журнала реализованы репозитории и основные операции доступа к данным.

#### 1) Кафедры (`lecterns`)
Репозиторий: `LecternRepository`

Добавлены методы:
- `Create` — добавить кафедру
- `GetByID` — получить кафедру по id
- `Update` — обновить код/название кафедры
- `Delete` — удалить кафедру
- `List` — получить список кафедр

Назначение:
- ведение справочника кафедр, к которым привязываются дисциплины, группы и преподаватели.

#### 2) Типы контроля (`control_types`)
Репозиторий: `ControlTypeRepository`

Добавлены методы:
- `Create` (upsert) — создать/обновить тип контроля
- `GetByID` — получить по id
- `Update` — изменить название
- `Delete` — удалить тип
- `List` — список типов

Назначение:
- хранение и управление вариантами контроля (экзамен, зачет и т.д.) для учебных планов.

#### 3) Предметы (`subjects`)
Репозиторий: `SubjectRepository`

Добавлены методы:
- `Create` — создать дисциплину
- `GetByID` — получить дисциплину
- `Update` — обновить дисциплину
- `Delete` — удалить дисциплину
- `List` — получить все дисциплины
- `ListByLecternID` — получить дисциплины конкретной кафедры

Назначение:
- база дисциплин, которые участвуют в расписании, контролях и ведомостях.

#### 4) Метрики предмета (`subject_metrics`)
Репозиторий: `SubjectMetricRepository`

Добавлены методы:
- `Upsert` — создать/обновить метрики дисциплины
- `GetBySubjectID` — получить метрики по `subject_id`
- `DeleteBySubjectID` — удалить метрики

Назначение:
- хранение учебной нагрузки по предмету (часы, ЗЕТ, план/факт), что важно для отчетности кафедры.

#### 5) Нагрузка по семестрам (`semester_load`)
Репозиторий: `SemesterLoadRepository`

Добавлены методы:
- `Create` — добавить запись нагрузки
- `GetByID` — получить запись по id
- `ListBySubjectID` — список нагрузки предмета по семестрам
- `Delete` — удалить запись

Назначение:
- детализация нагрузки дисциплины по семестрам для планирования учебного процесса.

#### 6) Формы контроля по семестрам (`subject_controls`)
Репозиторий: `SubjectControlRepository`

Добавлены методы:
- `Create` — добавить форму контроля к предмету
- `GetByID` — получить запись
- `ListBySubjectID` — список форм контроля по предмету
- `Delete` — удалить запись

Назначение:
- хранение того, какие проверки и в каком семестре проходят студенты.

#### 7) Преподаватели (`teachers`)
Репозиторий: `TeacherRepository`

Добавлены методы:
- `Create` — добавить преподавателя
- `GetByID` — получить по id
- `Update` — обновить данные
- `Delete` — удалить преподавателя
- `List` — список преподавателей
- `ListByLecternID` — преподаватели кафедры

Назначение:
- справочник преподавателей для привязки к дисциплинам и будущей фиксации посещаемости занятий.

#### 8) Группы (`groups`)
Репозиторий: `GroupRepository`

Добавлены методы:
- `Create` — создать группу
- `GetByID` — получить группу
- `Update` — обновить группу
- `Delete` — удалить группу
- `List` — список групп
- `ListByLecternID` — группы кафедры

Назначение:
- хранение учебных групп, с которыми далее связываются студенты и ведомости посещаемости.

#### 9) Студенты (`students`)
Репозиторий: `StudentRepository`

Добавлены методы:
- `Create` — добавить студента
- `GetByID` — получить по id
- `Update` — обновить данные студента
- `Delete` — удалить студента
- `List` — список студентов
- `ListByGroupID` — список студентов конкретной группы

Назначение:
- база студентов для формирования списков на пару и будущих журналов присутствия.
