
### Основной компонент
- **Файл:** `src/components/TeacherAccount.jsx`
- **Стили:** `src/components/TeacherAccount.css`

### API интеграция
- **Файл:** `src/services/api.js`
- **Методы:**
  - `createAttendanceLink()` - создание ссылки на посещаемость
  - `getGroupStats()` - получение статистики группы

## Функциональность

### 1. Создание Сессии Посещаемости

**Форма содержит поля:**
- `Subject ID` - ID предмета
- `Group IDs` - ID групп (через запятую)
- `Lesson name` - название урока
- `Expires minutes` - время действия ссылки (минуты)

**Результат:**
- Join URL - ссылка для студентов
- Invite token - токен приглашения
- Session ID - ID сессии
- Timestamp создания

**Поток:**
```
Преподаватель заполняет форму 
  → Клик "Создать ссылку" 
    → API запрос к `/api/teacher/attendance-link`
      → Получение URL и токена
        → Вывод результата с возможностью копирования
```

### 2. Статистика Группы

**Форма содержит поля:**
- `Group ID` - ID группы (обязательное)
- `Subject ID` - ID предмета (опционально)

**Результат:**
Таблица с данными по каждому студенту:
- `student_id` - ID студента
- `student_name` - ФИ студента
- `attended` - количество посещений
- `total` - всего занятий
- `%` - процент посещаемости

**Поток:**
```
Преподаватель вводит Group ID 
  → Клик "Загрузить посещаемость"
    → API запрос к `/api/teacher/attendance-stats`
      → Получение данных посещаемости
        → Вывод таблицы статистики
```

## Интеграция в приложение

### Роутинг (App.jsx)
```javascript
if (userData?.role === 'teacher') {
  return <TeacherAccount userData={userData} onLogout={handleLogout} token={token} />
}
```

При входе преподавателя автоматически отображается кабинет вместо студенческого интерфейса.

## Состояние и управление

### State переменные
- `sessionForm` - данные формы создания сессии
- `statsForm` - данные формы статистики
- `sessionResult` - результат создания сессии
- `statsResult` - результат запроса статистики
- `sessionLoading`, `statsLoading` - индикаторы загрузки
- `sessionError`, `statsError` - сообщения об ошибках
- `activeTab` - активная вкладка (sessions/statistics)

### Обработчики
- `handleSessionInputChange()` - изменение полей формы сессии
- `handleStatsInputChange()` - изменение полей формы статистики
- `handleCreateSession()` - создание сессии
- `handleGetStats()` - получение статистики

## UI/UX особенности

### Навигация
- Табы для переключения между функциями
- Активная вкладка подсвечивается синим цветом

### Безопасность
- Все запросы содержат Bearer token
- Токен передается в Authorization header

### Обработка ошибок
- Отображение сообщений об ошибках
- Отключение кнопок во время загрузки
- Очистка ошибок при новом запросе

### Адаптивность
- Резиновый дизайн (responsive)
- Мобильная оптимизация (grid меняется на одну колонну)
